# 3. 홈 페이지
## 가. 개요
### 1) 기획 의도
* 홈 페이지는 방문자의 첫 접점으로, 6개 섹션을 통해 "관심 → 신뢰 → 전환"의 흐름을 설계
* 전체 흐름: 가치 제안(Hero) → 기술 차별화 → 결과물 입증 → 서비스 분기 → 신뢰 보강 → 전환 유도

### 2) 페이지 구성

| 섹션 | AIDA 단계 | 핵심 역할 |
|------|-----------|-----------|
| Hero | Attention | 영상 배경 + 슬로건 + 신뢰 지표로 시선을 잡고 가치 제안 |
| AI 기술 차별점 | Interest | "왜 가능한지"를 기술력으로 설명하여 관심을 구체화 |
| 대표 작업물 | Interest → Desire | 실제 결과물로 기술력을 입증하여 욕구 전환 |
| 서비스 분기 | Desire | 두 서비스를 비교 제시하여 "나에게 맞는 서비스" 선택욕 자극 |
| 인사이트 프리뷰 | Desire | 전문 콘텐츠로 신뢰를 보강하여 결정 허들 낮춤 |
| 클로징 CTA | Action | 최종 전환 유도 |

## 나. 섹션 구성
### 1) Hero
* 영상 배경 위에 브랜드명·슬로건을 배치하여 AI 영상 제작 역량을 시각적으로 즉시 전달
* 신뢰 지표(500+ 프로젝트, 98% 만족도, 3x 속도)를 함께 노출하여 첫 화면에서 신뢰 확보
* CTA 2개(프로젝트 문의·포트폴리오 보기)로 전환 의향이 높은 방문자와 탐색형 방문자를 동시에 수용

### 2) AI 기술 차별점
* 개요의 차별화 전략("고퀄리티 결과물 이외에 AI 기술력 강조")을 홈에서 직접 실행하는 섹션
* 4개 기술 항목을 카드 형태로 제시하여 "이 퀄리티가 왜 가능한지"를 구체적으로 설명
* Hero에서 제시한 가치 제안의 근거를 기술력으로 뒷받침

### 3) 대표 작업물
* 기술 차별점 섹션 직후에 배치하여 "기술 → 결과물"의 인과 관계를 자연스럽게 연결
* 6개 포트폴리오를 3컬럼 그리드로 노출하고, 전체 보기 링크로 Work 페이지 유입 유도
* 결과물(What)을 통해 앞서 설명한 기술력(How)을 실증

### 4) 서비스 분기
* Production(외주 서비스)과 Studio(SaaS 솔루션) 두 서비스를 양분 카드로 나란히 제시
* 각 서비스별 체크리스트와 CTA를 분리하여 방문자가 자신에게 맞는 서비스를 선택하도록 유도
* "어떤 서비스가 맞는지 모르겠다면" 문의 CTA를 하단에 추가 배치하여 이탈 방지

### 5) 인사이트 프리뷰
* 최신 아티클 3개를 노출하여 AI 영상 분야의 전문성과 기술 리더십을 전달
* 서비스 분기 이후 전환을 결정하지 못한 방문자에게 신뢰를 보강하는 역할
* 전체 보기 링크로 Insights 페이지 유입 유도

### 6) 클로징 CTA
* 페이지 최하단에서 최종 전환을 유도하는 마감 섹션
* 프로젝트 문의 CTA와 포트폴리오 보기 링크를 다시 제공하여 전환 기회를 재확보

# 4. 영상 제작(Production) 페이지
## 가. 개요
### 1) 기획 의도
* AI 영상 제작 외주 서비스의 전용 랜딩 페이지
    * 전문가에게 맡기는 서비스로 포지셔닝
* 홈 페이지의 서비스 분기에서 영상 제작을 선택한 방문자가 유입되므로, 이미 관심이 있는 상태를 전제로 삼음
* 7개 섹션으로 "기술 신뢰 → 기존 대비 우위 입증 → 협업 프로세스 투명화 → 전환"의 설득 흐름을 설계
### 2) 페이지 구성
| 섹션 | AIDA 단계 | 핵심 역할 |
|------|-----------|-----------|
| 가치 제안 | Attention | 서비스 핵심 메시지와 즉시 문의 CTA 제공 |
| AI 기술 적용 | Interest | 자사 AI 기술이 제작에 어떻게 적용되는지 구체화 |
| 기존 vs AI 비교 | Interest → Desire | 기존 제작 방식 대비 품질·속도·비용 우위를 대조 |
| 대표 작업물 | Desire | Production 결과물만 선별 노출하여 퀄리티 실증 |
| 제작 프로세스 | Desire | 6단계 워크플로를 공개하여 협업 과정의 투명성 확보 |
| 전문가 구성 | Desire | 5개 직군의 전문 인력 제시로 조직 역량 신뢰 보강 |
| 클로징 CTA | Action | 최종 전환 유도 (Production 문의) |

## 나. 섹션 구성
### 1) 가치 제안 (Hero)
* 서비스명과 핵심 설명을 간결하게 제시하여 방문자가 올바른 페이지에 도착했음을 인지하도록 보조
* 프로젝트 문의 CTA를 첫 화면에 배치하여 전환 의향이 높은 방문자를 즉시 수용

### 2) AI 기술 적용
* 개요의 차별화 전략("고퀄리티 결과물 이외에 AI 기술력 강조")을 Production 맥락에서 구체 실행
* 4개 기술 항목을 제시하며 각 항목에 정량 수치를 포함하여 기술력의 체감도 향상

### 3) 기존 vs AI 비교 (Paradigm Shift)
* 기존 영상 제작 방식과 자사 AI 제작 방식을 동일 항목 기준으로 나란히 대조
* 자사 항목에 brand 컬러 강조를 적용하여 시각적으로 우위를 즉시 인지하도록 유도
* "이 퀄리티가 왜 가능한지"에 대한 답을 비교 구조로 제시

### 4) 대표 작업물
* 영상 제작 작업물만 필터링하여 6개 노출 — 서비스에 특화된 포트폴리오 제시
* 전체 보기 링크로 Work 페이지 유입 유도 
    * 쿼리 파라미터 기반 필터 연동 기능 고려

### 5) 제작 프로세스
* 6단계 스텝을 카드로 순서대로 배치하여 협업 과정을 투명하게 공개
* 의뢰부터 납품까지의 흐름을 시각화하여 기업 고객의 의사결정 불확실성 해소

### 6) 전문가 구성
* 5개 직군(크리에이티브 디렉터, AI 엔지니어 등)을 카드로 제시하여 조직 전문성 전달
* "AI만으로 만드는 것이 아니라 전문가가 AI를 활용한다"는 메시지로 품질 신뢰 보강

### 7) 클로징 CTA
* 페이지 최하단에서 Production 문의로의 최종 전환 유도
* type=production 파라미터를 전달하여 Contact 페이지에서 문의 유형이 자동 선택

# 5. AI 스튜디오(Studio) 페이지
## 가. 개요
### 1) 기획 의도
* AI 영상 생성 솔루션의 전용 랜딩 페이지
    * 직접 만드는 도구 제공으로 포지셔닝
* 6개 섹션으로 "제품 이해 → 기술 기반 신뢰 → 기능 확인 → 도입 효과 체감 → 전환"의 설득 흐름을 설계
* SaaS 제품 특성상 기능·효과를 구체적으로 제시하여 도입 의사결정을 지원

### 2) 페이지 구성

| 섹션 | AIDA 단계 | 핵심 역할 |
|------|-----------|-----------|
| 제품 Hero + UI 프리뷰 | Attention | 제품 핵심 메시지 전달 + UI 미리보기로 제품 형태 인지 |
| 엔진 차별점 | Interest | 자사 AI 엔진의 기술적 우위를 구체화 |
| 핵심 기능 | Interest → Desire | 주요 기능을 카드로 나열하여 활용 가능성 확인 |
| 비즈니스 임팩트 | Desire | 정량 수치로 도입 효과를 체감시켜 욕구 자극 |
| 활용 사례 | Desire | Studio 결과물을 실제 사례로 입증 |
| 클로징 CTA | Action | 최종 전환 유도 (Studio 도입 문의) |

## 나. 섹션 구성
### 1) 제품 Hero + UI 프리뷰
* 서비스명과 핵심 설명을 제시하고 도입 문의 CTA를 첫 화면에 배치
* Hero 직하단에 브라우저 프레임 형태의 제품 UI 스크린샷 배치로 SaaS 제품임을 시각적으로 즉시 인지
* Production 페이지와 달리 "제품 화면"을 보여줌으로써 도구형 서비스라는 차별적 포지셔닝을 강화

### 2) 엔진 차별점
* 개요의 차별화 전략("AI 기술력 강조")을 Studio 맥락에서 실행
* 4개 기술 항목을 제시하여 자사 AI 엔진이 타 솔루션과 어떻게 다른지 설명
* Production의 "AI 기술 적용" 섹션과 대응하되, Studio에서는 엔진 자체의 성능에 초점

### 3) 핵심 기능
* AI 아바타, 자동 자막, 다국어 더빙 등 주요 기능을 6개 카드로 나열
* 기업 고객이 "이 제품으로 무엇을 할 수 있는지"를 구체적으로 파악할 수 있도록 지원
* 관심(Interest)에서 욕구(Desire)로 전환하는 핵심 섹션

### 4) 비즈니스 임팩트
* 도입 효과를 정량 수치(비용 절감률, 제작 시간 등)로 제시하여 ROI 체감
* 기능 나열만으로는 부족한 의사결정자 설득을 수치 기반으로 보완
* 컬러로 수치를 강조하여 시각적 임팩트 부여

### 5) 활용 사례
* Studio 스타일 작업물만 필터링하여 6개 노출 — 솔루션으로 제작한 결과물 실증
* 전체 보기 링크로 Work 페이지 유입 유도 
    * 쿼리 파라미터 기반 필터 연동 기능 고려

### 6) 클로징 CTA
* 페이지 최하단에서 Studio 도입 문의로의 최종 전환 유도
* type=studio 파라미터를 전달하여 Contact 페이지에서 문의 유형이 자동 선택

# 6. 포트폴리오(Work) 페이지
## 가. 개요
### 1) 기획 의도
* 서비스 결과물의 퀄리티를 실제 사례로 입증하는 페이지
* Production·Studio 양쪽 서비스의 결과물을 한 곳에서 탐색할 수 있도록 통합 구성
* 다단계 필터(카테고리 → 산업군·스타일·목적)로 방문자가 자신과 관련된 사례를 빠르게 찾도록 지원
* 3개 섹션의 단순 구조로 "탐색 → 발견 → 전환"에 집중

### 2) 페이지 구성

| 섹션 | AIDA 단계 | 핵심 역할 |
|------|-----------|-----------|
| Hero | Attention | 페이지 진입 확인 및 포트폴리오 소개 |
| 필터 + 그리드 | Interest → Desire | 다단계 필터로 관련 사례 탐색, 결과물로 퀄리티 입증 |
| 클로징 CTA | Action | 프로젝트 문의로 전환 유도 |

## 나. 섹션 구성
### 1) Hero
* 페이지 제목과 설명으로 포트폴리오 페이지임을 간결하게 전달
* 별도 CTA 없이 바로 필터·그리드로 이어지는 탐색 중심 구조

| 컴팩트 히어로 + 신뢰 지표 | Attention | 문의 페이지 도착 인지 + 신뢰 수치로 최종 확신 보강 |
| 문의 폼 (좌측) | Action | 서비스 유형별 맞춤 폼으로 문의 접수 |
| 사이드바 (우측) | Desire → Action | 선택 이유·상담 프로세스·FAQ·오피스 정보로 의사결정 지원 |

## 나. 섹션 구성
### 1) 컴팩트 히어로 + 신뢰 지표
* 간결한 헤딩과 부제로 문의 페이지의 목적을 즉시 전달
* 하단에 신뢰 지표(trust stats)를 숫자 카드로 나란히 배치
    * 문의 직전 단계에서 정량적 성과 수치를 다시 제시하여 전환 결심의 최종 근거 제공
* 어두운 배경(slate-950)으로 히어로 영역을 시각적으로 분리

### 2) 문의 폼
* Production / Studio 탭 선택으로 문의 유형을 전환
    * URL 파라미터(type)가 있으면 해당 탭이 자동 선택되어 유입 맥락 유지
* 탭 전환 시 탭별 베네핏 카피를 표시하여 해당 서비스의 핵심 가치를 상기
* ContactFormFields 공통 컴포넌트로 탭별 폼 필드를 동적 구성
    * Production: 회사명, 담당자명, 이메일, 전화번호, 목적, 마감일, 예산, 레퍼런스, 메시지, 파일 첨부
    * Studio: 회사명, 담당자명, 이메일, 전화번호, 팀 규모, 월 생성량, 활용 목적, 필요 기능, 메시지, 파일 첨부
* 동일 컴포넌트를 FloatingCTA 모달에서도 재사용하여 폼 일관성 확보
* 제출 완료 시 성공 화면(체크 아이콘 + 메시지)으로 전환

### 3) 사이드바
* **Why Loomix AI**: 자사 선택 이유를 체크리스트로 나열하여 전환 직전 설득 보강
* **상담 프로세스**: 문의 접수 후 진행 단계를 스텝 카드로 시각화하여 다음 절차의 불확실성 해소
* **FAQ 바로가기**: 자주 묻는 질문 페이지로 연결하여 문의 전 궁금증 자체 해결 경로 제공
* **오피스 정보**: 주소, 업무시간, 이메일, 전화번호를 아이콘과 함께 나열하여 실재하는 기업임을 전달
* **Google Maps**: 오피스 위치를 지도로 표시하여 물리적 접근성과 신뢰 보강
