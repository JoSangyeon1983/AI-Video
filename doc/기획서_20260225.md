# 1. 개요

## 1.1. 프로젝트 목적

AI를 이용한 **하이엔드 영상 제작 에이전시 서비스**와 **기업용 AI 영상 생성 SaaS 솔루션**을 홍보하는 브랜드 웹사이트를 개발한다. 웹사이트는 B2B 의사결정권자를 대상으로 자사의 기술력을 증명하고, 문의 전환(리드 확보)을 극대화하는 것을 핵심 목표로 한다.

**핵심 요구사항:**
- SEO(검색엔진최적화), AEO(AI 엔진 최적화), GEO(생성형 AI 최적화)를 동시에 충족
- SSG(Static Site Generation) 기반 정적 빌드로 빠른 로딩 속도 확보
- 한국어·영어·일본어 3개 언어 지원
- 포트폴리오(Work) 중심의 실력 증명 + 듀얼 CTA(Service/Solution) 전환 구조

## 1.2. 브랜드 정의 (Loomix AI)

| 항목 | 값 |
|------|-----|
| 브랜드명 | **Loomix AI** |
| 브랜드 설명 | 하이엔드 AI 영상 제작 에이전시 및 영상 생성 솔루션 |
| 브랜드 슬로건 | 하이엔드 퀄리티를, 더 빠르고 더 유연하게. |
| 도메인 | https://loomix.ai |

- 브랜드 상수는 `src/data/brand.ts`에서 중앙 관리되며, 변경 시 전체 사이트에 자동 반영된다.

## 1.3. 타겟 고객 및 비즈니스 모델

### 비즈니스 이원화 (Business Bifurcation)

| 구분 | Service (에이전시) | Solution (SaaS) |
|------|-------------------|-----------------|
| 모델 | DIFM (전문가 전담 제작) | DIY (고객 직접 사용) |
| 타겟 | 하이엔드 영상을 외주하려는 B2B 기업 | 반복 영상을 내재화하려는 마케팅 팀 |
| 핵심 가치 | 퀄리티·속도·유연성 극대화 | 비용 절감·대량 생산·팀 자체 운영 |
| 대표 스타일 | 실사, 3D, 모션그래픽 | AI 아바타 |
| CTA 컬러 | Brand (Electric Blue) | Secondary (Violet) |
| 문의 경로 | `/contact/?type=service` | `/contact/?type=solution` |

### GNB 배치 전략

```
[로고] [Work] [Service] [Solution] [Insights] [Story] [문의]  [🌐]
        ──── 시각적 증명 ───  ── 비즈니스 분기 ──  ── 권위 구축 ──   i18n
```

1. **시각적 증명 우선(Proof-First):** Work를 첫 번째 메뉴로 배치하여 결과물로 즉각 신뢰 확보
2. **비즈니스 모델 이원화:** Service(용역)와 Solution(SaaS)을 명확히 분리
3. **기능적 레이어링:** Contact는 텍스트 링크, 실질적 전환은 Floating CTA가 전담
4. **권위(Authority) 구축:** Insights와 Story를 후반부에 배치하여 전문성 및 SEO 트래픽 확보

## 1.4. 핵심 전략 (SEO · AEO · GEO)

| 전략 | 구현 내용 |
|------|----------|
| **SEO** | 페이지별 Metadata · canonical URL · sitemap.xml · robots.txt · OG/Twitter Card · 시맨틱 HTML |
| **AEO** | Schema.org JSON-LD (Organization, VideoObject, Article, FAQPage) · 구조화된 콘텐츠(H1→H2 계층) |
| **GEO** | hreflang(ko/en/ja/x-default) · 3개 언어 번역 · 구조화 데이터의 다국어 지원 |

## 2.3. 빌드 및 배포

### Next.js 설정 (`next.config.ts`)

```typescript
const nextConfig: NextConfig = {
  output: "export",        // SSG 정적 빌드 (out/ 디렉토리 출력)
  trailingSlash: true,     // URL 끝에 / 보장 (/work/ 형태)
  images: { unoptimized: true },  // 정적 빌드 시 이미지 최적화 비활성
  devIndicators: false,
};
```

- `output: "export"`: 모든 페이지를 빌드 시점에 HTML로 정적 생성
- `trailingSlash: true`: SEO 일관성을 위해 모든 URL 끝에 `/` 보장

### Express 정적 서버 (`server/server.js`)

| 기능 | 설명 |
|------|------|
| 정적 파일 서빙 | `front/out/` 디렉토리의 빌드 결과물을 Express.static으로 서빙 |
| trailing slash 리다이렉트 | `/path` → `/path/` (301 리다이렉트) |
| HTML 캐시 방지 | HTML 파일에 `no-cache, no-store, must-revalidate` 헤더 적용 |
| 폴백 라우팅 | 존재하지 않는 경로는 `/index.html` → `/path/index.html` → `/path.html` 순서로 탐색 |
| 자동 브라우저 오픈 | 서버 시작 시 `http://localhost:3000` 자동 열기 |

### 3.1.1. 데스크톱 네비게이션 (1024px 이상)

```
[로고(이미지)] [Work] [Service] [Solution] [Insights] [Story]  [문의]  [🌐 한국어 ▾]
```

| 요소 | 설명 |
|------|------|
| 로고 | `Image` 컴포넌트, 140×32px, 다크모드 시 `brightness-0 invert` 적용 |
| 메뉴 항목 | `navItems` 배열 기반 렌더링 (Work / Service / Solution / Insights / Story) |
| 문의 링크 | `/contact/`로 이동하는 텍스트 링크 (GNB 메뉴와 시각적으로 분리) |
| 언어 선택기 | `LanguageSelector` 드롭다운 (ko/en/ja) |

### 3.1.2. 모바일 햄버거 메뉴 (1023px 이하)

- 햄버거 아이콘(☰) 클릭 시 **풀스크린 오버레이** 메뉴 전개
- 메뉴 항목: Home + 전체 GNB 메뉴를 세로 배치
- 하단 CTA 영역:
  - "제작 의뢰" 버튼 → `/contact/?type=service`
  - "솔루션 도입" 버튼 → `/contact/?type=solution`
- 메뉴 열림 시 `useBodyScrollLock`으로 배경 스크롤 방지
- 메뉴 항목 클릭 시 자동 닫힘

### 3.1.3. 스크롤 시 스타일 변화

```
스크롤 Y > 10px:
  ├─ 배경: bg-white/90 → backdrop-blur-md 적용
  ├─ 하단 보더: border-slate-200 추가
  └─ 그림자: shadow-sm 추가
```

- `useEffect`에서 `scroll` 이벤트를 감지하여 `scrolled` 상태 토글
- 투명 → 불투명 전환으로 콘텐츠와의 시각적 분리 명확화

### 3.1.4. 현재 페이지 Active 표시

- `isActivePath(pathname, href)`: 현재 경로와 메뉴 href를 비교
- 일치 시: `bg-brand-600 text-white` (활성 스타일)
- 불일치 시: `text-slate-600 hover:bg-slate-100` (비활성 스타일)
- 하위 경로도 매칭 (예: `/work/beauty-campaign`에서 Work 메뉴 활성)

## 4.2. I18nProvider 및 Context 구조

> **파일:** `src/i18n/context.tsx`

```
I18nContext
├── locale: Locale          // 현재 언어 코드
├── t: Translation          // 현재 언어 번역 객체
└── setLocale(locale)       // 언어 변경 함수
```

**동작 흐름:**
1. 초기 렌더링: `ko` 번역을 동기적으로 로드 (번들에 포함)
2. `localStorage`에서 `loomix-locale` 키를 확인하여 저장된 언어 복원
3. 언어 변경 시:
   - `localStorage`에 저장
   - `document.documentElement.lang` 동기화
   - 해당 언어 번역 파일을 `dynamic import`로 로드
   - 로드 완료 후 `translations` 상태 업데이트

## 4.4. Lazy Import 전략

```typescript
const translationMap: Record<Locale, () => Promise<Translation>> = {
  ko: () => Promise.resolve(ko),           // 동기 — 번들에 포함
  en: () => import("./translations/en"),    // 비동기 — 필요 시 로드
  ja: () => import("./translations/ja"),    // 비동기 — 필요 시 로드
};
```

- **ko (한국어):** 기본 언어이므로 번들에 동기 포함 → 초기 로딩 지연 없음
- **en/ja:** `dynamic import()`로 필요 시에만 로드 → 초기 번들 크기 절감
- 한국어 유저(대다수)는 추가 네트워크 요청 없이 즉시 사용 가능

# 10. 인터랙션 · 모션 설계

## 10.1. Hero 진입 애니메이션

| 순서 | 요소 | 효과 | 딜레이 |
|------|------|------|--------|
| 1 | H1 (브랜드명) | opacity 0→1, y 30→0 | 0s |
| 2 | Sub copy (슬로건) | opacity 0→1, y 20→0 | 0.2s |
| 3 | CTA 버튼 | opacity 0→1, y 20→0 | 0.4s |
| 4 | Trust 지표 컨테이너 | opacity 0→1 | 0.7s |
| 5 | Trust 각 항목 | opacity 0→1, scale 0.8→1 | 0.8s + i×0.15s |

- 모든 모션: `duration 0.5~0.8s`, `ease: [0.25, 0.1, 0.25, 1]` (EASE_OUT)

## 10.2. 스크롤 기반 Reveal

| 사용처 | 방향 | distance | duration |
|--------|------|----------|----------|
| 섹션 제목 (H2) | up | 40px | 0.6s |
| Business Card A | left | 40px | 0.6s |
| Business Card B | right | 40px | 0.6s |
| Timeline 항목 | 좌우 교차 | 30px | 0.6s |

- `viewport.once: true` → 한 번만 실행 (재스크롤 시 반복하지 않음)
- `viewport.amount: 0.15` → 요소의 15% 진입 시 트리거

## 10.3. StaggerContainer 순차 등장

| 사용처 | stagger | 항목 수 |
|--------|---------|---------|
| Featured Work 그리드 | 0.1s | 6 |
| Feature 카드 | 0.08s | 6 |
| Process Stepper | 0.1s | 6 |
| Expert 카드 | 0.08s | 5 |
| Impact 카드 | 0.12s | 4 |
| 팀원 프로필 | 0.1s | N |
| 파트너 로고 | 0.06s | N |

## 10.4. 필터/그리드 전환 애니메이션

- `AnimatePresence mode="wait"`: 그리드 전체가 exit → enter 순차 전환
- exit: `opacity 0, y -10`
- enter: `opacity 0→1, y 20→0`
- 개별 카드: `delay: idx * 0.06s` stagger

## 10.5. 호버 인터랙션

| 요소 | 호버 효과 |
|------|----------|
| VideoCard | 썸네일 `scale(1.05)` + 프리뷰 영상 재생 + 재생 아이콘 표시 + `shadow-lg` |
| Business Card | `shadow-xl` |
| Feature Card | `shadow-lg` |
| Article Card | `shadow-md` |
| Floating CTA | `scale(1.05)` + `shadow-xl` |

## 10.6. Floating CTA 버튼 Shine 이펙트

```css
.btn-shine::before {
  content: '';
  background: linear-gradient(90deg, transparent, rgba(255,255,255,0.4), transparent);
  animation: shine 3s infinite;
  transform: skewX(-15deg);
}
@keyframes shine {
  0%   { left: -100%; }
  50%, 100% { left: 150%; }
}
```

- 3초 간격으로 반복되는 빛 이펙트로 CTA 주목도 강화
- `overflow: hidden` + `z-index: 10` + `pointer-events: none`으로 클릭 방해 없음

**11.1. Work 정렬:** 현재 필터링은 업종/스타일/목적으로 가능하나, 최신순/인기순/추천순 정렬 기능이 없음. 정렬 드롭다운 UI 추가 필요.
