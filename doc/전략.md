# Part 1. 홈 페이지

---

## 1. 차별화 포지셔닝

경쟁사 대부분은 **영상 결과물(포트폴리오)** 을 전면에 내세우는 전략을 취한다.
Loomix AI는 **자체 AI 기술(LX Engine) + 결과물**을 함께 내세워,
"이 퀄리티가 왜 가능한지"까지 설명하는 **기술 기반 차별화** 포지셔닝을 구축한다.

이 전략이 홈 페이지 섹션 순서에 직접 반영되어야 한다.

---

## 2. 현재 섹션 구성 (확정)

| # | 섹션 | AIDA 단계 | 핵심 역할 |
|---|------|-----------|-----------|
| 1 | **Hero** | Attention | 브랜드명 + 가치 제안 + 듀얼 CTA + Trust 수치 |
| 2 | **AI Technology Edge** | Interest | LX Engine 기술 차별화 선언 (Tell) |
| 3 | **Featured Work** | Desire | 기술로 완성한 결과물 증명 (Show) |
| 4 | **Business Bifurcation** | Desire → Action | 에이전시 vs SaaS 서비스 자기 선택 유도 |
| 5 | **Insights Preview** | Trust | AI 기술 리더십 권위 보강 |
| 6 | **Closing CTA** | Action | 최종 전환 ("무료 상담 신청") |
| 상시 | **FloatingCTA** | Action | 우하단 고정 버튼 (제작 문의 / 스튜디오 문의) |

---

## 3. 섹션 순서 설계 근거

### 3-1. Tell → Show (기술 → 결과물)

- **섹션 2(Tech Edge) → 섹션 3(Featured Work)** 순서를 채택
- 기술을 먼저 보여주면 이후 포트폴리오를 볼 때 **프레이밍 효과** 작동
  - "AI 기술로 만들었기 때문에 이 퀄리티가 가능하구나"라는 인과 추론 유도
- 경쟁사가 결과물만 보여줄 때, Loomix는 "왜 가능한지"까지 설명하는 것이 차별점
- B2B 의사결정자는 퀄리티뿐 아니라 **지속적 제작 역량(=기술)** 을 평가함

### 3-2. Insights → Closing CTA (권위 보강 → 최종 전환)

- Closing CTA를 **페이지의 마지막 콘텐츠 섹션**으로 배치 (Footer 직전)
- Insights가 Closing CTA 앞에서 "이 회사는 AI 영상 분야의 전문가"라는 권위 신호 보강
- 전환하지 않고 나가려는 유저도 Closing CTA를 반드시 지나가게 됨
- CTA 이후 추가 콘텐츠가 없으므로 "더 볼 게 없다" → 전환 결정 촉진

### 3-3. CTA 밀도 최적화

- FloatingCTA가 항상 떠 있으므로, 중간 섹션마다 인라인 CTA를 넣으면 화면에 3개씩 보이는 구간 발생
- 과도한 CTA는 "공격적 판매" 인상, 각 CTA의 긴급성 희석, 시각적 노이즈 유발
- **전환 CTA는 3채널로 정리**: Hero(상단) + Closing CTA(하단) + FloatingCTA(상시)
- 중간 섹션의 CTA는 "전체 포트폴리오 보기" 등 **탐색 링크**만 유지

# Part 2. 영상 제작(Production) 페이지

## 1. 차별화 전략

홈 페이지가 "LX Engine의 4가지 핵심 역량"을 **선언**하는 역할이라면,
Production 페이지는 **"제작 현장에서 LX Engine이 구체적으로 어떻게 쓰이는지"**를 보여주는 역할이다.

경쟁사 제작 서비스 페이지는 대부분 "포트폴리오 + 가격/속도" 구조를 취한다.
이 구조에서는 AI 기술력이 드러나지 않으므로, **기술 전용 섹션을 별도로 신설**하여 차별화한다.

---

## 2. 현재 섹션 구성 (확정)

| # | 섹션 | AIDA 단계 | 핵심 역할 |
|---|------|-----------|-----------|
| 1 | **Hero** | Attention | AI 기반 가치 제안 + CTA ("제작 문의") |
| 2 | **AI Tech Applied** | Interest | LX Engine이 제작에 적용되는 4가지 방식 (Tell) |
| 3 | **Paradigm Shift** | Interest → Desire | 기존 제작 vs Loomix 수치 비교 (Prove) |
| 4 | **Featured Works** | Desire | 기술로 완성한 결과물 6건 (Show) |
| 5 | **Process** | Trust | 6단계 전문가 주도 AI 제작 프로세스 |
| 6 | **Experts** | Trust | 5인 전문가 팀 구성 |
| 7 | **Closing CTA** | Action | 최종 전환 ("제작 문의하기") |

---

## 3. 섹션 순서 설계 근거

### 3-1. Tell → Prove → Show (기술 → 수치 → 결과물)

- **섹션 2(AI Tech) → 섹션 3(Paradigm Shift) → 섹션 4(Featured Works)** 3단계 설득 구조 채택
- 홈 페이지의 Tell→Show 2단계를 확장하여, 중간에 **정량적 수치 증명** 단계를 삽입
- 흐름: "이런 기술이 있다"(기술 카드) → "그래서 이런 수치가 나온다"(비교 테이블) → "실제로 이런 결과를 냈다"(포트폴리오)
- 기술을 먼저 보여준 뒤 비교 테이블을 읽으면, 수치가 "왜 가능한지" 인과관계가 자동으로 형성됨

### 3-2. Process → Experts (워크플로우 → 팀)

- 프로세스를 먼저 보여주면 "AI와 전문가가 각 단계에서 어떻게 협업하는가"가 드러남
- 이어서 전문가 팀을 보여주면 "이 프로세스를 실행하는 사람들"로 자연스럽게 연결
- AI만으로 하는 게 아니라 **"전문가가 AI를 무기로 쓰는 것"**이라는 메시지 전달

### 3-3. CTA 밀도 최적화 (홈 페이지 전략과 동일)

- 전환 CTA는 **Hero(상단) + Closing CTA(하단) + FloatingCTA(상시) = 3채널**만 유지
- 중간 섹션에는 인라인 CTA를 배치하지 않아 과도한 CTA 노출 방지
- Featured Works의 "더 많은 포트폴리오 보기"는 탐색 링크로 유지

## 4. 각 섹션 상세 전략

### 섹션 1: Hero

| 항목 | 내용 |
|------|------|
| 헤딩 | "AI 기반 영상 제작 서비스" + "기획부터 하이엔드 편집까지" |
| 설명 | 전문가 주도 + LX Engine 역할 명시 |
| CTA | Primary "제작 문의" (→ /contact/?type=production) |

### 섹션 2: AI Tech Applied (신규)

| 항목 | 내용 |
|------|------|
| 구성 | 4카드 그리드 (아이콘 + 제목 + 정량 수치 뱃지 + 설명) |
| 카드 1 | 🧬 브랜드 파인튜닝 — "초안 정확도 78%" |
| 카드 2 | 🔄 1소스→N포맷 자동 생성 — "포맷 변환 시간 90% 단축" |
| 카드 3 | ✅ AI 품질 자동 검증 — "브랜드 일관성 96%" |
| 카드 4 | 📈 성과 연동 최적화 — "CTR 평균 1.6배 향상" |
| CTA | 없음 (다음 섹션 비교 테이블로 자연스러운 흐름 유도) |

**핵심 포인트**
- 홈 페이지 Tech Edge가 "LX Engine이 뭘 할 수 있는가"라면, 이 섹션은 "제작 현장에서 구체적으로 어떻게 쓰이는가"
- 각 카드에 정량 수치(stat 뱃지)를 포함하여 추상적 기술 설명이 아닌 실증 데이터로 설득

**보완 사항**
- Before/After 비교 비주얼: "일반 AI 생성 결과 vs LX Engine 파인튜닝 결과" 스크린샷/영상 비교 삽입
- 기술 파트너 로고: OpenAI, RunwayML, Google Cloud 등 파트너 로고 배치로 공신력 보강
- 특허/인증 배지: Story 타임라인의 "LX Engine 특허 출원" 사실을 이 섹션에도 표시

### 섹션 3: Paradigm Shift (비교 테이블)

| 항목 | 내용 |
|------|------|
| 구성 | "기존 제작" vs "Loomix 제작" 2컬럼 비교 카드 |
| 비교 항목 | 제작 리드타임 (4~8주 → 1~2주), 제작 비용 (높음 → 최대 60% 절감), 수정 리드타임 (5~7일 → 1~2일), 투입 리소스 (대규모 팀 → 소규모 전문 팀) |
| 시각 강조 | Loomix 쪽에 border-brand-600 브랜드 컬러 보더 + bold 텍스트 |

**보완 사항**
- 미니 케이스 스니펫 삽입: 비교 테이블 하단에 실제 성과 1~2줄 요약 (예: "D2C 뷰티 A사: 수정 요청 3.2회→0.6회, 제작 10일→3일")
- 벤치마크 비교 행 추가: "초안 사용 가능률" (Sora 34% / Runway 41% / LX Engine 78%) 등 인사이트 아티클 데이터 발췌

### 섹션 4: Featured Works

| 항목 | 내용 |
|------|------|
| 구성 | VideoCard 6건 (에이전시/하이엔드 필터, AI 아바타 제외) |
| 목적 | AI Tech + Paradigm Shift에서 제시한 기술/수치의 실물 결과 확인 |
| CTA | "더 많은 포트폴리오 보기" 탐색 링크 (→ /work/?style=agency) |

**보완 사항**
- 각 VideoCard에 "적용 기술" 태그 표시 (예: "Brand Finetuning", "1→N 자동 생성")
- 클라이언트 로고 배너를 포트폴리오 상단에 배치하여 Social Proof 강화

### 섹션 5: Process

| 항목 | 내용 |
|------|------|
| 구성 | 6단계 스텝 카드 (브리프 접수 → 기획/콘티 → 룩디자인 → 제작/AI생성 → 편집/후반 → 납품) |
| 목적 | AI와 전문가의 협업 워크플로우 투명하게 공개 |
| Layout | 2×3 그리드 (xl: 6컬럼) |

**보완 사항**
- 각 단계에 AI/Human 역할 분리 시각화: Step 1~2 🧑 전문가, Step 3~4 🤖+🧑 협업, Step 5~6 🧑 전문가
- "AI가 다 하는 게 아니라 전문가가 AI를 무기로 쓰는 것"이라는 핵심 메시지 시각적으로 전달

### 섹션 6: Experts

| 항목 | 내용 |
|------|------|

## 3. 섹션 순서 설계 근거

### 3-1. Tell → Show → Prove → Evidence (기술 → 기능 → 수치 → 사례)

- **섹션 2(Engine) → 섹션 3(Features) → 섹션 4(Impact) → 섹션 5(Cases)** 4단계 설득 구조 채택
- Production의 Tell→Prove→Show 3단계를 확장하여, SaaS 의사결정에 필요한 **ROI 수치**를 별도 단계로 분리
- 흐름:
  1. "이 스튜디오는 어떤 기술 위에 만들어졌나" (Engine Advantage — 기반 기술)
  2. "그래서 구체적으로 무엇을 할 수 있나" (Features — 기능)
  3. "도입하면 어떤 숫자가 나오나" (Impact — ROI)
  4. "실제로 이렇게 쓰이고 있다" (Use Cases — 실증)

### 3-2. Impact를 Features 뒤로 이동한 이유

- 기존 순서: Impact(2) → Features(3) — 수치를 먼저 보여줌
- 변경 순서: Features(3) → Impact(4) — 기능을 이해한 뒤 수치를 봄
- "어떤 기능인지도 모르면서 80% 단축이라 해도 납득이 안 됨"
- 기능을 먼저 이해한 상태에서 수치를 보면 → **"이 기능이면 80% 단축 가능하겠다"** 자연스러운 납득
- Impact에는 인라인 CTA 없이 수치만 제시 → Use Cases → Closing CTA로 자연스럽게 이어지도록 설계

### 3-3. CTA 밀도 최적화

- **Hero(상단) + Closing CTA(하단) + FloatingCTA(상시) = 3채널**
- 기존에는 Impact 섹션에 인라인 CTA("스튜디오 문의")가 있어 4채널이었으나,
  Closing CTA와 **동일한 링크(/contact/?type=studio)로 중복**되고,
  섹션 4→5→6 짧은 구간에 CTA 2개가 연속 배치되어 전환 압박감이 과도했음
- 제거 후 Impact 수치가 순수한 증거 역할에 집중하며,
  Use Cases(5)를 거쳐 Closing CTA(6)에서 모든 설득을 완료한 뒤 한 번에 전환 유도
- Production 페이지(3채널)와 동일한 밀도로 사이트 전체 일관성 확보

## 3. 섹션 순서 설계 근거

### 3-1. 리스트 페이지 — 기술 태그가 포함된 갤러리

- Hero 카피에 "LX Engine 기반 AI 기술로 완성한 포트폴리오"를 명시하여, 갤러리 진입 전에 **기술 프레이밍** 설정
- 각 VideoCard에 업종/스타일/목적 태그 아래 **techTags 뱃지**(Brand Finetuning, 1→N 자동 생성 등)를 추가 표시
- 카드를 훑어보는 것만으로 "각 작품에 다른 AI 기술이 적용되었구나"를 인식 → **기술 다양성과 실전 적용 역량** 증명
- techTags는 Production 계열(`bg-brand`)과 Studio 계열(`bg-secondary`)로 색상 분리 → 서비스 라인 시각적 구분

### 3-2. 디테일 페이지 — Tell → Prove (기술 → 케이스 스터디)

- **섹션 3(적용된 AI 기술) → 섹션 4(Case Detail)** 순서 채택
- 기술 뱃지를 케이스 스터디 앞에 배치하면, Challenge/Approach/Result를 읽을 때 **"이 기술이 적용되었기 때문에 이 결과가 가능했구나"** 인과 추론 유도
- 홈 → Production → Studio에서 반복한 **Tell → Prove 패턴**을 디테일 페이지에서도 일관 적용
- 적용된 AI 기술 섹션은 카드 형태(`rounded-xl border`)로 시각적 분리 → 케이스 스터디 텍스트와 충돌 방지

### 3-3. CTA 밀도 최적화

- **리스트 페이지**: Closing CTA(하단) + FloatingCTA(상시) = 2채널
  - Hero에는 CTA 버튼 없음 (리스트 페이지의 목적은 탐색이므로 전환 압박 최소화)
  - 갤러리를 끝까지 스크롤한 유저에게만 Closing CTA 노출
- **디테일 페이지**: Detail CTA(하단) = 1채널
  - FloatingCTA는 work/[slug] 경로에서 비활성 (기존 설정 유지)
  - 케이스 스터디를 다 읽은 유저에게만 CTA → "이 퀄리티로 만들고 싶다" 전환 최적 타이밍
  - CTA 버튼에 `?type={production|studio}&ref={slug}` 쿼리 → 상담 시 맥락 자동 전달

## 3. 섹션 순서 설계 근거

### 3-1. 리스트 페이지 — 기술 프레이밍 → 콘텐츠 탐색

- **Hero에 sub 카피 추가**: "LX Engine 개발팀이 직접 공유하는 AI 영상 기술과 비즈니스 전략"
  - 경쟁사 블로그 Hero는 "우리의 인사이트"로만 표현 → Loomix는 **"기술 보유사의 1차 정보"** 프레이밍
  - 다른 페이지들과 동일한 "LX Engine 먼저 언급" 패턴 유지
- **기술 하이라이트 스트립을 Hero 직후에 배치**: 아티클을 하나도 읽지 않아도 핵심 기술 수치를 인식
  - 96% 브랜드 일관성 · 78% 초안 사용률 · 6분 30초 영상 생성 · 80% 수정 감소
  - 홈 페이지 Tech Edge, Production AI Tech Applied와 동일한 "기술 수치 먼저 선언" 패턴
- **techBadge로 기술 관련 아티클 시각적 구분**: 방문자가 "기술력을 보고 싶다"면 brand 컬러 뱃지가 붙은 글을 즉시 식별

### 3-2. 디테일 페이지 — 콘텐츠 소비 → 전환

- 목차 → 섹션 본문 → CTA → 관련 아티클 순서로, **콘텐츠를 다 읽은 시점에 CTA 노출**
- CTA가 본문 직후에 위치하여 "이 기술을 우리도 적용하고 싶다" 전환 타이밍 포착
- 관련 아티클을 CTA 뒤에 배치하여, 전환하지 않은 유저도 **추가 콘텐츠로 재유입** → 체류 시간 증대
- ctaType에 따라 Production/Studio CTA 자동 분기 → 아티클 내용과 맥락이 맞는 CTA 제시

### 3-3. CTA 밀도 최적화

- **리스트 페이지**: Closing CTA(하단) + FloatingCTA(상시) = 2채널
  - Hero에 CTA 없음 → 인사이트 페이지의 목적은 "콘텐츠 소비"이므로 전환 압박 최소화
  - 아티클을 훑어본 후 Closing CTA에서 "더 알고 싶다면 문의"로 자연스러운 전환
- **디테일 페이지**: Detail CTA(본문 하단) + FloatingCTA(상시) = 2채널
  - 본문 중간에 CTA 삽입 없음 → 읽기 흐름 방해 방지
  - CTA 링크에 `?type={ctaType}&ref={slug}` → 상담 시 "어떤 아티클을 읽고 문의했는지" 맥락 전달

# Part 6. 스토리(Story) 페이지

## 1. 차별화 전략

스토리 페이지는 브랜드의 **비전·연혁·팀**을 소개하는 "About Us" 역할을 한다.
경쟁사 대부분의 About 페이지는 창업 스토리와 팀 소개에 머물며,
"왜 이 회사의 기술이 다른가"에 대한 정량적 증거를 제시하지 않는다.

Loomix AI는 스토리 페이지에 **"기술 역량 & 성과(Tech Capabilities)"** 섹션을 독립 배치하여,
"이 팀이 만든 기술이 어떤 성과를 내고 있는가"를 **수치와 구조화된 카드**로 보여준다.

이 전략의 핵심 메커니즘:

1. **Hero 카피**에 "자체 AI 엔진(LX Engine)"을 명시 → 다른 페이지와 동일한 기술 프레이밍 패턴 유지
2. **Timeline에서 성장사**를 보여준 직후, **Tech Capabilities에서 기술 성과를 정량화** → 성장 서사 + 기술 증거의 시너지
3. **Team 섹션**에서 기술 전문가 비중이 높다는 점을 부각 → 기술 역량의 인적 기반 증명

# Part 7. 문의하기(Contact) 페이지

---

## 1. 차별화 전략

문의하기 페이지는 사이트 전체의 **최종 전환 목적지**이다.
다른 모든 페이지의 CTA(Hero CTA, Closing CTA, FloatingCTA)가 이 페이지로 수렴한다.

경쟁사 대부분의 Contact 페이지는 **폼만 나열**하는 구성으로,
앞선 페이지에서 쌓아온 기술 차별화 인식이 폼 진입 시점에서 **단절**된다.

Loomix AI는 Contact 페이지에 세 가지 기술 프레이밍 장치를 삽입하여,
**"AI 기술 전문 회사에 문의하고 있다"**는 인식을 폼 작성 중에도 지속시킨다:

1. **Compact Hero** — LX Engine 프레이밍 + Trust 수치 스트립으로 기술 맥락 선언
2. **탭별 Benefit 카피** — 탭 전환 시 해당 서비스의 AI 기술 혜택을 즉시 안내
3. **Why Loomix AI 체크리스트 + 상담 프로세스** — 우측 Sidebar에서 폼 작성 중에도 기술 근거 상시 노출

이 전략의 핵심은 **폼 포기율 감소**이다. 기술 신뢰 요소가 폼 근처에 없으면 작성 도중 이탈이 증가하지만,
Trust 수치와 기술 체크리스트가 시야에 계속 들어오면 **"맡길 만한 회사"** 확신이 유지된다.

## 2. 현재 섹션 구성 (확정)

| # | 섹션 | AIDA 단계 | 핵심 역할 |
|---|------|-----------|-----------|
| 1 | **Compact Hero** _(신규)_ | Attention | LX Engine 기술 프레이밍 + Trust 수치 4개 |
| 2-L | **Form (좌측)** | Action | 탭(Production/Studio) + Benefit 카피 + 폼 필드 + 제출 |
| 2-R | **Sidebar (우측)** _(강화)_ | Trust → Action | Why Loomix AI 체크리스트 + 상담 프로세스 + FAQ 링크 + 오피스 정보 + 지도 |
| - | **Success 화면** | Confirmation | 접수 완료 + 1영업일 내 연락 안내 |

> Contact 페이지는 **전환 목적 단일 페이지**이므로, 섹션을 다수 나열하기보다
> **기존 2컬럼 구조 안에서 기술 프레이밍과 Trust 요소를 주입**하는 것이 적합하다.

특이사항:
- **FloatingCTA 숨김**: Contact 페이지에서는 `FloatingCTA` 컴포넌트가 명시적으로 `return null` 처리됨
  - 이미 폼이 핵심 전환 수단이므로, 추가 CTA는 시각적 노이즈
- **PageHero 미사용**: 다른 페이지와 달리 커스텀 Compact Hero 사용 — 폼까지의 스크롤 거리 최소화

## 3. 섹션 순서 설계 근거

### 3-1. Compact Hero — 기술 맥락 연속성

- **헤딩**: "AI 영상 전문가에게 직접 상담받으세요"
- **서브카피**: "LX Engine 기반 맞춤 제안 — 프로젝트 목표에 최적화된 AI 영상 솔루션을 설계합니다"
- **Trust 수치 스트립**: `500+건 완료` · `96% 브랜드 일관성` · `1영업일 내 응답` · `2주 평균 납기`

**설계 근거:**
- 다른 페이지에서 CTA 클릭 → Contact 진입 시, **"여기서도 AI 기술 회사에 문의하고 있다"**는 연속성 유지
- 홈·Production·Studio·Work·Insights·Story 모두 Hero에서 LX Engine을 언급 → Contact도 동일 패턴 유지
- Trust 수치가 폼 상단에 위치하여, "이 회사는 실적이 있다" → **폼 작성 결심 촉진**
- 풀 PageHero가 아니라 **Compact 스트립**이므로, 폼까지의 스크롤 거리 최소화 (전환 페이지 특성)

### 3-2. Form + Sidebar 2컬럼 — 작성 중 신뢰 유지

- 좌측은 폼에 집중, 우측은 폼을 작성하면서 시야에 들어오는 **보조 설득 콘텐츠**
- 이 구조는 B2B SaaS 업계에서 검증된 패턴:
  - HubSpot, Salesforce, Notion 등 → 좌측 폼 + 우측 Trust 요소

**좌측 (Form) 설계:**
- 탭 전환으로 Production/Studio 폼 필드 자동 분기
- **탭별 Benefit 카피**(brand 컬러 배경)가 탭 직후에 노출 → "이 서비스를 선택하면 어떤 기술 혜택을 받는가" 즉시 인식

**우측 (Sidebar) 설계:**
- **Why Loomix AI**: 4항목 체크리스트(brand 컬러 아이콘) → 폼 작성 중 "왜 이 회사인가"의 기술 근거 상시 노출
- **상담 프로세스**: 3단계 카드 → "문의하면 어떤 일이 일어나는가" 프리뷰 → 불확실성 제거 → 전환률 상승
- **FAQ 바로가기**: 폼 작성 전 궁금증 해소 → 이탈 방지
- **오피스 정보 + 지도**: 실재 기업 확인 → B2B 신뢰 최종 보강

### 3-3. URL 파라미터를 통한 맥락 전달

- 다른 페이지 CTA에서 `/contact/?type=production` 또는 `/contact/?type=studio`로 유입 시, 해당 탭이 자동 선택됨
- `useSearchParams`로 `type` 파라미터 감지 → `setPurpose(type)` 자동 적용
- 사용자가 Production 페이지에서 CTA 클릭 → Contact 진입 시 이미 Production 탭이 선택되어 있음 → 맥락 연속성

### 3-4. CTA 밀도 최적화

- **폼 자체가 CTA**이므로 추가 CTA 불필요
- FloatingCTA가 Contact 페이지에서 명시적 숨김 → 시각적 노이즈 제거
- 제출 버튼 1개만 존재 → **단일 액션 집중**

## 6. 전체 사이트 일관성 — 기술 레이어링 패턴 (업데이트)

| 페이지 | 기술 섹션 | 설득 구조 | 기술 노출 수준 |
|--------|----------|-----------|---------------|
| **홈** | Tech Edge (4카드) | Tell → Show | 선언 (기술이 존재한다) |
| **Production** | AI Tech Applied (4카드) | Tell → Prove → Show | 적용 (제작에 이렇게 쓰인다) |
| **Studio** | Engine Advantage (4카드) | Tell → Show → Prove → Evidence | 아키텍처 (제품의 기반이다) |
| **Work 리스트** | techTags 뱃지 (카드 내) | Browse → Discover | 증거 (각 작품에 적용되었다) |
| **Work 디테일** | 적용된 AI 기술 + Case Detail | Tell → Prove | 검증 (기술→성과 인과관계) |
| **Insights 리스트** | 기술 하이라이트 스트립 + techBadge | Summarize → Browse | 요약 (핵심 수치 + 콘텐츠 식별) |
| **Insights 디테일** | 아티클 본문 (데이터 포함) | Educate → Convert | 심층 (기술 원리와 실증 데이터) |
| **Story** | Tech Capabilities (4수치+4카드) | Narrate → Prove → Trust | 역량 (보유 기술과 성과 정량화) |
| **Contact** | **Hero Trust + Why체크리스트 + 프로세스** | **Trust → Convert** | **전환 (기술 신뢰 유지 × 폼 작성 촉진)** |

**핵심 원칙**: Contact 페이지는 기술 레이어링의 **최종 수렴지**이다.
다른 페이지에서 쌓아온 기술 차별화 인식을 **폼 작성 순간까지 끊기지 않게** 하는 것이 Contact의 유일한 역할이다.
새로운 기술 정보를 추가로 제공하기보다, 이미 인지한 핵심 수치(500+, 96%, 6분)를 **재확인**시키는 것에 집중한다.
